Прогнозирование Deepfake Voice для выявления синтетической речи
Цель проекта — построить модель, которая определяет, является ли аудиозапись настоящей человеческой речью или сгенерированной (deepfake voice). 
​​В этом проекте я работаю с датасетом Deep Voice – Deepfake Voice Recognition с Kaggle и строю модель, которая определяет, является ли аудиозапись реальной человеческой речью или синтетически сгенерированной. Такая система может использоваться для выявления голосового мошенничества, защиты биометрических систем и проверки подлинности аудиоконтента. 
На первом этапе я провожу исследовательский анализ данных: изучаю распределение классов (real/fake), длительность записей, частотные характеристики и визуализирую спектрограммы. Это позволяет увидеть различия между настоящей и синтетической речью, обнаружить возможные артефакты и понять структуру аудиоданных. 
Далее выполняю извлечение признаков. Из аудио формируются MFCC, Mel-спектрограммы, спектральная энергия, zero-crossing rate и другие характеристики. При необходимости добавляются агрегированные статистики по временным окнам. Эти признаки помогают модели уловить тонкие акустические различия между реальным и сгенерированным голосом.
Для обучения я использую как классические алгоритмы (XGBoost, LightGBM), так и нейросетевые подходы например, CNN на спектрограммах. Модель решает задачу бинарной классификации. Качество оценивается с помощью метрик AUC-ROC, F1-score и Accuracy. 
Отдельно проверяется устойчивость модели: как она ведёт себя на новых записях, зашумлённом аудио и данных другого распределения. Это важно, потому что методы генерации deepfake постоянно развиваются, и система должна быть устойчивой к новым типам атак. 
В завершении модель упаковывается в Docker и разворачивается через FastAPI. Это позволяет передавать аудиофайл через API и получать вероятность того, что голос является deepfake. При необходимости можно добавить простой интерфейс для демонстрации работы системы.
Датасет (например, на основе Kaggle)
● Audio file — аудиозапись
● Label — real / fake
● Speaker ID — идентификатор спикера
● Duration — длительность
● Sample Rate — частота дискретизации
● Codec / Recording type — тип записи
● Augmentation flags — применялась ли генерация/модификация
Идеальный формат данных: аудио + табличные метаданные.
Метрики
● AUC-ROC (основная)
● F1-score
● Accuracy
● EER (Equal Error Rate — важно для биометрии)
Этапы проекта
1. Анализ данных
Сначала проводится анализ:
распределение real vs fake
длительность записей
спектрограммы
сравнение частотных характеристик
выявление артефактов синтетической речи
Это помогает понять структуру данных и потенциальные паттерны deepfake
2. Feature Engineering
Из аудио извлекаются признаки: MFCC, Mel-spectrogram, Chroma features, Spectral contrast, Zero-crossing rate, Energy
3. Моделирование
Классические модели: XGBoost, LightGBM, Random Forest
Deep Learning: CNN на спектрограммах, CNN + LSTM, Transformer-based модели, Fine-tuning wav2vec
Оценка через стратифицированную кросс-валидацию.
4. Проверка устойчивости
Важно проверить: устойчивость к новым типам deepfake, дрейф распределений, переобучение на конкретные голоса, работу на зашумлённых записях. Это критично для реального применения.
5. Продакшн
Модель оборачивается в Docker.
Вход: аудиофайл
Выход: вероятность deepfake, итоговый класс (real/fake)
Инструменты

Инфраструктура: Jupyter Notebook, Docker, FastAPI
